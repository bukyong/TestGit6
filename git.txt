* git은 폴더(디렉토리)단위로 해당 파일을 관리
	=> 특정 폴더를 정보를 저장하는 저장소(repository) 설정

* untracked 상태 : git이 관리 대상에서 제외한 상태(파일)
* tracked 상태 : git이 관리 대상으로 설정한 상태(파일)

(tracked 상태 이후 확인 가능)
unstaged 상태 : 관리 대상이기는 하나 아직 이력 상태를 저장하지 않은 경우(제외)
staged 상태 : 관리 대상이면서 이력 상태를 저장한 경우 (commit)

* 저장소 생성 => git init

> git staturs : 상태 확인

> git add : 파일의 상태를 tracked 상태로 변경

> git에 사용자 정보 등록
  git config --global user.email "you@example.com"
  git config --global user.name "Your Name"
